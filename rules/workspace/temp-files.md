# 임시 파일 관리 지침

작업 중 생기는 임시·스크래치 파일을 **어디에 두고, 버전 관리에서 어떻게 제외하며, 언제 정리하는지**를 정하는 규칙입니다. 임시 파일이 리포 곳곳에 흩어지거나 실수로 커밋되는 것을 막습니다.

## 적용 범위

이 지침은 **임시·스크래치 파일**에만 적용됩니다. 예: 한 번 쓰고 버리는 실험 스크립트, 디버깅 로그, 중간 다운로드, 탐색용 메모, 일회성 출력 덤프.

- 영구 산출물·소스 코드·생성된(generated) 코드·릴리스 빌드 결과물의 위치는 이 지침이 규정하지 않습니다. 그것은 엔지니어링(빌드 산출물 위치) 또는 소스 트리 규약의 영역입니다.
- "이 파일을 나중에도 쓸 것인가?"가 애매하면 임시가 아닙니다 — 정식 위치에 둡니다.

## 저장 위치 (필수 규칙)

**모든 임시·스크래치 파일은 프로젝트 트리 내부의 전용 디렉터리 `.tmp/` 아래에 둔다.**

- 프로젝트 루트의 `.tmp/`를 기본 임시 영역으로 삼습니다. 하위 작업별로 `.tmp/<주제>/`처럼 나눌 수 있습니다.
- 시스템 임시 디렉터리(`/tmp`·`%TEMP%` 등)나 프로젝트 밖 경로를 기본 저장 위치로 삼지 않습니다. 임시 산출물도 작업 맥락과 함께 한곳에서 보이고 정리되도록 프로젝트 안에 둡니다.
- 소스 디렉터리·리포 루트에 임시 파일을 흩뿌리지 않습니다. 루트에 `scratch.py`, `output.txt` 같은 파일을 직접 만들지 않습니다.
- 디렉터리 이름을 바꿔야 하면 이 규칙의 `.tmp/`를 일괄로 고쳐 단일 규약을 유지합니다.

## 버전 관리 제외 (필수 규칙)

**`.tmp/` 전체를 버전 관리에서 제외한다.**

- `.gitignore`에 `.tmp/` 항목을 등록해 임시 디렉터리의 내용이 추적·커밋되지 않게 합니다.
- 임시 파일은 리뷰·릴리스·히스토리에 남지 않아야 합니다. `git add .` 등으로 실수로 들어왔다면 커밋 전에 제거합니다.
- 디렉터리 존재 자체를 공유해야 하면 `.tmp/.gitkeep`만 추적하고 나머지는 무시하는 식으로 좁힙니다(선택).
- 이 규칙은 규율이며, `.gitignore`를 자동으로 편집하지 않습니다 — 등록은 작업자가 직접 합니다.

## 정리·수명 (필수 규칙)

**임시 파일은 사용 목적이 끝나면 정리한다. 임시 영역에 무기한 쌓아두지 않는다.**

- 한 작업(태스크·이터레이션·세션)이 끝나면 그 작업이 만든 `.tmp/` 산출물을 정리합니다.
- 다음 단계에서 다시 쓸 임시 파일은 남겨도 되지만, 더 이상 참조되지 않으면 지웁니다.
- 어떤 임시 파일이 사실은 보존 가치가 있다고 판단되면, 그것은 임시가 아닙니다 — `.tmp/` 밖 정식 위치로 옮기고 적절히 버전 관리합니다.
- 이 규칙은 규율이며, 자동 정리 스크립트나 강제 훅을 만들지 않습니다 — 정리는 작업자가 직접 합니다.

## 위반 발견 시

임시 파일이 `.tmp/` 밖에 있거나, `.tmp/`가 `.gitignore`에 없거나, 임시 산출물이 커밋·릴리스에 섞여 들어간 것을 발견하면 즉시 멈추고 정상화합니다 — 파일을 `.tmp/`로 옮기거나 추적에서 제외합니다. 다음 작업으로 미루지 않습니다.
